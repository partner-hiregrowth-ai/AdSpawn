AdSpawn — CEO Demo Flow (~10 min)
=====================================

BEFORE THE DEMO
- Make sure you have at least 2-3 live campaigns in your Meta ad account
- Best if one is OUTCOME_TRAFFIC (for the conversion demo later)
- Have the app running: backend on localhost:5000, frontend on localhost:3000


STEP 1 — Login (30s)
---------------------
1. Open https://localhost:3000
2. Click "Login with Facebook"
3. Authorize > lands on Dashboard

Say: "This connects directly to your Meta Business account via OAuth."


STEP 2 — Explorer: Browse Live Campaigns (1 min)
-------------------------------------------------
1. Click "Explorer" in the sidebar
2. Select your ad account from the dropdown
3. Click a campaign row to expand it > shows ad sets
4. Click an ad set > shows ads underneath

Say: "This is a live mirror of what's on Meta right now.
      You can see the full hierarchy — campaign, ad sets, ads — all in one view.
      In Ads Manager this takes 10+ clicks across different screens."


STEP 3 — Duplicate a Campaign to Draft (30s)
---------------------------------------------
1. In Explorer, find a TRAFFIC campaign (e.g. "OUTCOME_TRAFFIC Campaign 01")
2. Click the "Duplicate" button on that campaign row
3. Wait for the success toast

Say: "One click — the entire campaign structure is cloned locally.
      Campaign settings, all ad sets with their targeting, all ads with their creatives.
      Nothing is touched on Meta yet."


STEP 4 — Edit the Draft (2 min)
--------------------------------
1. Click "Drafts" in the sidebar
2. Open the campaign you just duplicated
3. You are on the Campaign level:
   - Change the campaign name (e.g. add " - Q3 Test" at the end)
   - Change Bid Strategy dropdown to "Cost Per Result Goal"
   - Notice it now shows a bid amount field (dynamic form reacts to your choices)

4. Click an Ad Set in the left tree panel:
   - Change the ad set name
   - Change Optimization Goal to a different value
   - Edit a targeting field (e.g. age_min)

5. Click back to the Campaign in the tree:
   - Point out: "The ad set changes were auto-saved when I switched.
     No save button needed — you never lose work when navigating the tree."

6. Click an Ad in the tree:
   - Change the ad name
   - Show the Creative ID field and inline creative fields

7. Click the "Advanced" tab:
   - Show the full schema-driven form
   - "Every field Meta supports for this objective is here,
     dynamically generated — no hardcoded forms."

Say: "You edit everything in one place. Campaign, ad sets, ads — all on one screen.
      The form adapts to the objective. Fields that don't apply are hidden automatically."


STEP 5 — Validate Before Publishing (1 min)
--------------------------------------------
1. Click the "Validate" button (shield icon)
2. If validation passes: show the green success toast

   Say: "Before we talk to Meta's API, we run the same constraint checks locally.
        Bid strategy needs a bid amount? Caught. Lifetime budget without an end date? Caught.
        Messenger placement without Facebook? Caught. Thailand targeting below age 20? Caught.
        All in plain English — not cryptic API error codes."

3. (Optional) To show a validation error on purpose:
   - Set Bid Strategy to "Cost Per Result Goal" but clear the bid amount
   - Click Validate again
   - Show the error: "Cost Per Result Goal requires a bid amount."
   - Then fix it and re-validate to show it passes


STEP 6 — Publish to Meta (1 min)
---------------------------------
1. Click "Publish" button
2. Confirm the dialog: "Publish to Meta? This will create real campaigns, ad sets, and ads (all PAUSED)."
3. Wait for the success toast (takes ~5-10 seconds)

4. Go back to Explorer
5. Refresh the ad account
6. Find the new campaign — it's there on Meta, status PAUSED

Say: "Real campaign on Meta. Created in seconds. Everything starts PAUSED —
      nothing goes live until you manually activate it in Ads Manager.
      Zero risk of accidental spend."


STEP 7 — Convert Objective: The Power Feature (2 min)
------------------------------------------------------
This is the key differentiator. Meta does NOT allow changing a campaign's objective.

1. Go to Explorer
2. Find the TRAFFIC campaign you duplicated earlier (the original, not the copy)
3. Click "Convert Objective" on that campaign
4. In the dialog, select "Sales" as the new objective
5. Wait for it to process

6. Go to Drafts, open the converted campaign
7. Point out what changed automatically:
   - Objective: OUTCOME_TRAFFIC -> OUTCOME_SALES
   - Optimization Goal: LINK_CLICKS -> OFFSITE_CONVERSIONS
   - Destination Type: WEBSITE (kept, valid for Sales)
   - Promoted Object: NOW has pixel_id field (Sales requires it)
   - Attribution Spec: NOW included (only valid for Sales/Leads/App)

Say: "In Meta Ads Manager, this is impossible. You'd have to manually create
      a new campaign, recreate every ad set with new settings, copy every ad,
      and figure out which fields are valid for the new objective.

      We do it in one click. The system knows the rules:
      - Sales needs a pixel_id, so it's added
      - The optimization goal is migrated to the closest equivalent
      - Invalid fields are stripped, required fields are filled
      - Everything is validated before you publish

      This saves hours of manual work and eliminates human error."


STEP 8 — Bulk Operations (30s)
-------------------------------
1. Go to Drafts list
2. Check the boxes on 2-3 campaigns
3. Click "Bulk Publish"
4. Show all of them publishing in one batch

Say: "When you're managing 50-100 campaigns, you publish them all at once.
      Each one is individually validated and error-handled.
      If one fails, the others still succeed — you get a clear report of what happened."


STEP 9 — Wide Creation (1 min, if time permits)
------------------------------------------------
1. Click "Wide Create" in the sidebar
2. Choose a template or create new:
   - 1 Campaign (Traffic)
   - 3 Ad Sets (different targeting per set)
   - 2 Ads per ad set
3. Configure in the tree view
4. Click Generate

Say: "6 ads generated from one template. Same structure, different variations.
      For A/B testing at scale — instead of building each one manually."


=== END OF DEMO ===

Key Messages to Reinforce Throughout
--------------------------------------
1. SPEED:    "What takes 30 minutes in Ads Manager takes 30 seconds here"
2. SAFETY:   "Every publish is validated first. Errors in plain English, not API codes"
3. UNIQUE:   "Objective conversion is impossible natively — this is our differentiator"
4. SCALE:    "One campaign or a hundred — same workflow"
5. CONTROL:  "Everything starts PAUSED. Zero risk of accidental spend"


Promoted Object Requirements (reference)
-----------------------------------------
| Objective        | Requires promoted_object? | Field needed                      |
|------------------|---------------------------|-----------------------------------|
| AWARENESS        | No                        | —                                 |
| TRAFFIC          | No                        | —                                 |
| ENGAGEMENT       | No                        | —                                 |
| LEADS            | Yes                       | page_id                           |
| SALES            | Yes                       | pixel_id + custom_event_type      |
| APP_PROMOTION    | Yes                       | application_id                    |

Best conversion demo paths:
- Traffic -> Sales  (shows pixel_id being added)
- Traffic -> Leads  (shows page_id being added)
- Awareness -> Traffic (simple, safe, no promoted_object needed)
